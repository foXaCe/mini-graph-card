## Findings considered and rejected

- **`graph.ts` getPath/getPoints mutate `this.coords` on single-point series**
  (`coords[1] = …` on an aliased reference): real smell but reachable only for a
  1-sample series, well inside 85%-covered battle-tested code; not worth the
  regression risk for the payoff. Revisit only if a single-point bug is reported.
- **`Math.min(...this.coords.map())` spread (graph.ts:108-109)**: theoretical
  call-stack risk only for pathologically large point counts; buckets are bounded
  by `hours_to_show * points_per_hour` (and bars by `MAX_BARS=96`). Not worth it.
- **`initialize.ts` low coverage (33%)**: side-effect module (cache purge + version
  banner); testing it well needs heavy localForage mocking for little safety gain.
- **`getObjectAttr` / dynamic attribute access staying `any`**: intentional
  looseness against HA's untyped `attributes` bag (see `types.ts` header comment);
  tightening it would add noise without catching real bugs. (Plan 004 deliberately
  leaves this one alone.)
